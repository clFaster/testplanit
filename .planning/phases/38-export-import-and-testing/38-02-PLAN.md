From testplanit/lib/llm/services/llm-manager.service.ts:
```typescript
async resolveIntegration(
  feature: string,
  projectId: number,
  resolvedPrompt?: { llmIntegrationId?: number; modelOverride?: string }
): Promise<{ integrationId: number; model?: string } | null>
```

Resolution chain in resolveIntegration:
1. Level 1: LlmFeatureConfig (projectId + feature unique lookup)
   - If found AND integration is active/not deleted -> return { integrationId, model }
2. Level 2: resolvedPrompt.llmIntegrationId (per-prompt assignment)
   - If provided AND integration is active/not deleted -> return { integrationId, model: resolvedPrompt.modelOverride }
3. Level 3: getProjectIntegration(projectId) -> project default
   - Falls back to system default if no project integration
4. Returns null if nothing found

Mock prisma models needed:
- llmFeatureConfig.findUnique (Level 1)
- llmIntegration.findUnique (Level 2 active check)
- projectLlmIntegration.findFirst (Level 3)
- llmProviderConfig.findFirst (system default fallback)
```
