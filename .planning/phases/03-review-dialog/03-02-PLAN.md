**TagChip.tsx** -- Toggleable tag chip with new/existing visual distinction and inline edit:

Props: `{ tagName: string; isExisting: boolean; isAccepted: boolean; onToggle: () => void; onEdit: (newName: string) => void; }`

- Default state: render a `Badge` component. When `isAccepted`, use variant "default" (filled). When not accepted, use variant "outline" with `opacity-50` class.
- New tag distinction (per user decision): when `!isExisting`, add `border-dashed` class and a small "New" label (`<span className="ml-1 text-[10px] font-normal opacity-70">New</span>`) inside the badge.
- Include `Tag` icon from lucide-react (w-3 h-3 mr-1) to match existing tag display patterns in the app.
- Click handler: calls `onToggle`.
- Double-click handler: enters edit mode. Replace badge with a small Input (`@/components/ui/input`). Auto-focus. On Enter or blur: call `onEdit(value)` and exit edit mode. On Escape: revert value and exit edit mode. Trim whitespace; if empty after trim, revert to original.
- Add `cursor-pointer select-none transition-all` classes for interactivity.

**EntityList.tsx** -- Left column of the two-column dialog:

Props: `{ entities: AutoTagSuggestionEntity[]; selectedEntityId: number | null; onSelectEntity: (entityId: number) => void; selections: AutoTagSelection; }`

- Text search input at top using `Input` component with `Search` icon from lucide-react. Filter entities by name (case-insensitive includes).
- Wrap the list in `ScrollArea` from `@/components/ui/scroll-area` for independent scrolling.
- Each entity row shows: entity name (truncated with ellipsis if long), and a count of accepted suggestions (e.g., "3 tags"). Show accepted count as `selections.get(entityId)?.size ?? 0`.
- Selected entity row has a highlighted background (e.g., `bg-accent`).
- If entity has zero suggestions (all tags array is empty), show a dimmed indicator.
- Click on row calls `onSelectEntity(entityId)`.

**EntitySuggestions.tsx** -- Right column showing suggestions for the selected entity:

Props: `{ entity: AutoTagSuggestionEntity; selections: AutoTagSelection; onToggle: (entityId: number, tagName: string) => void; onEdit: (entityId: number, oldName: string, newName: string) => void; }`

- Header: entity name as a subtitle.
- "Suggested Tags" section: Render each tag in `entity.tags` as a `TagChip`. Pass `isAccepted` from the selections Map (check if tagName is in the entity's set). Pass toggle/edit handlers bound to the entity's ID.
- "Current Tags" section (per user decision: "Show already-applied tags as non-interactive context in a separate grayed-out section"): If `entity.currentTags.length > 0`, show a labeled section with grayed-out badges (variant "secondary" with reduced opacity, no click handlers). Label: "Current Tags" or "Already Applied".
- If entity has no suggested tags (empty tags array): show a muted message like "No suggestions for this entity."
- Wrap in `ScrollArea` for independent scrolling of the right panel.

**AutoTagReviewDialog.tsx** -- Main dialog composing all sub-components:

Props: `{ open: boolean; onOpenChange: (open: boolean) => void; job: UseAutoTagJobReturn; }`

The dialog receives the hook return value from its parent (Phase 4 entry points will call `useAutoTagJob` and pass the return value). This keeps the dialog purely presentational.

- Use `Dialog` and `DialogContent` from `@/components/ui/dialog`. Override width: `className="max-w-[900px] h-[600px] flex flex-col"`. The dialog should have a fixed height to maintain the two-column layout stability.
- `DialogHeader` with title: "Review Tag Suggestions".
- Body: Two-column layout using CSS grid or flex. Left column ~35% width with `EntityList`. Right column ~65% width with `EntitySuggestions` for the selected entity.
- Manage `selectedEntityId` as local state (useState). Initialize to the first entity's ID when `job.suggestions` changes. Auto-select first entity on open.
- `DialogFooter` (sticky at bottom):
  - Left side: aggregate summary text. Use `job.summary` to display: "{existingCount} existing tags, {newCount} new tags will be created". If both are 0, show "No tags selected".
  - Right side: "Cancel" button (variant "outline", calls `onOpenChange(false)`) and "Apply" button (variant "default").
  - Apply button: disabled when `job.isApplying` or `job.summary.existingCount + job.summary.newCount === 0`. Show spinner/loading text when `isApplying`.
  - On Apply click: call `job.apply()`. On success: use `toast.success()` from sonner with message like "X tags applied to Y entities, Z new tags created" (compute from summary). Call `invalidateModelQueries` with the right model name (map entityType: `repositoryCase` -> `"RepositoryCases"`, `testRun` -> `"TestRuns"`, `session` -> `"Sessions"`). Also invalidate `"Tags"`. Then call `onOpenChange(false)` and `job.reset()`.
  - On Apply failure: use `toast.error()` with error message. Dialog stays open with selections preserved (per user decision).

- Use `useQueryClient()` from `@tanstack/react-query` for the invalidation call.

**AutoTagProgress.tsx** -- Inline progress indicator shown while job is processing:

Props: `{ status: AutoTagJobState; progress: { analyzed: number; total: number } | null; error: string | null; onReview: () => void; onCancel: () => void; }`

Per user decision: "While AI processes: inline progress indicator in the page (replaces trigger button area), not a dialog or toast."

- When `status === 'idle'`: render nothing (return null).
- When `status === 'waiting'` or `status === 'active'`:
  - Show a compact inline container with: text "Analyzing entities..." or "Analyzed {analyzed}/{total} entities", a `Progress` bar from `@/components/ui/progress` with value `(analyzed/total)*100`, and a "Cancel" text button.
  - If progress is null (waiting state before first poll), show indeterminate state (no percentage, just "Starting...").
- When `status === 'completed'`:
  - Per user decision: "When processing completes: success notification with 'Review' button to open the dialog."
  - Show: check icon, "Analysis complete" text, and a "Review Suggestions" button (calls `onReview`).
- When `status === 'failed'`:
  - Show: error icon, error message (or generic "Analysis failed"), and a dismiss/retry option.

Keep the component compact -- it replaces the trigger button area inline, not a full-width banner.
