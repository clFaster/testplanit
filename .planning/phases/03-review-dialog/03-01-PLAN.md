```typescript
import type { EntityType } from "~/lib/llm/services/auto-tag/types";

/** Shape of a single entity's suggestions as stored in job result */
export interface AutoTagSuggestionEntity {
  entityId: number;
  entityType: EntityType;
  entityName: string;
  currentTags: string[];
  tags: Array<{
    tagName: string;
    isExisting: boolean;
    matchedExistingTag?: string;
  }>;
}

/** Per-entity selection state: set of accepted tag names */
export type AutoTagSelection = Map<number, Set<string>>;

/** Job status from polling endpoint */
export type AutoTagJobState = 'idle' | 'waiting' | 'active' | 'completed' | 'failed';

/** Return type of useAutoTagJob hook */
export interface UseAutoTagJobReturn {
  // Job lifecycle
  jobId: string | null;
  status: AutoTagJobState;
  progress: { analyzed: number; total: number } | null;
  error: string | null;

  // Results and selections
  suggestions: AutoTagSuggestionEntity[] | null;
  selections: AutoTagSelection;

  // Tag edits: original name -> edited name
  edits: Map<string, string>;

  // Actions
  submit: (entityIds: number[], entityType: EntityType, projectId: number) => Promise<void>;
  toggleTag: (entityId: number, tagName: string) => void;
  editTag: (entityId: number, oldName: string, newName: string) => void;
  apply: () => Promise<void>;
  cancel: () => Promise<void>;
  reset: () => void;

  // Computed
  summary: { existingCount: number; newCount: number };
  isApplying: boolean;
  isSubmitting: boolean;
}
```

**File 2: `testplanit/components/auto-tag/useAutoTagJob.ts`**

Create the custom hook that manages the entire auto-tag workflow. Use `"use client"` directive at top.

Key implementation details:

1. **State:** `jobId` (string|null), `status` (AutoTagJobState starting as 'idle'), `progress` ({analyzed, total}|null), `suggestions` (AutoTagSuggestionEntity[]|null), `selections` (AutoTagSelection), `edits` (Map<string, string>), `error` (string|null), `isApplying` (boolean), `isSubmitting` (boolean).

2. **submit(entityIds, entityType, projectId):**
   - Set `isSubmitting = true`, `status = 'waiting'`, `error = null`
   - POST to `/api/auto-tag/submit` with `{ entityIds, entityType, projectId }`
   - On success, store `jobId` from response, which triggers polling
   - On failure, set `error`, `status = 'failed'`, `isSubmitting = false`
   - On success, set `isSubmitting = false`

3. **Polling via useEffect:**
   - Trigger when `jobId` is truthy AND `status` is 'waiting' or 'active'
   - Poll `GET /api/auto-tag/status/${jobId}` every 2 seconds (use setInterval)
   - On each poll: update `status` to match response `state`, update `progress` from response `progress`
   - When `state === 'completed'`: store `result.suggestions` in suggestions state, initialize `selections` map (all accepted by default using `initSelections` helper), clear interval
   - When `state === 'failed'`: set `error` from `failedReason`, clear interval
   - Cleanup: clearInterval on unmount or when polling stops
   - Do initial fetch immediately, then setInterval for subsequent

4. **initSelections(suggestions):** Creates a `Map<number, Set<string>>` where every entity maps to a Set of all its suggested tag names. This implements the "all accepted by default" decision.

5. **toggleTag(entityId, tagName):**
   - Clone selections Map, get or create Set for entityId
   - If tag is in set, delete it (reject); if not, add it (accept)
   - Update selections state

6. **editTag(entityId, oldName, newName):**
   - In selections: remove oldName from entity's set, add newName
   - In edits Map: set oldName -> newName (track for apply payload construction)
   - In suggestions state: update the tag's tagName in the entity's tags array (so UI reflects the edit)

7. **apply():**
   - Set `isApplying = true`
   - Build payload: iterate suggestions, for each entity get its accepted set from selections. For each accepted tagName, check edits map for a renamed version. Construct `{ entityId, entityType, tagName: finalName }` entries.
   - POST to `/api/auto-tag/apply` with `{ suggestions: payload }`
   - On success: set `isApplying = false`, return (let the caller handle dialog close and toast)
   - On failure: set `isApplying = false`, throw error (let caller show error toast while keeping dialog open)

8. **cancel():**
   - If `jobId`, POST to `/api/auto-tag/cancel/${jobId}`
   - Reset state to idle

9. **reset():** Clear all state back to initial values.

10. **summary (useMemo):** Compute from suggestions + selections. For each entity, for each tag in accepted set, check `isExisting` on the original suggestion tag. Count existing vs new. Handle edits: if a tag was edited, find the original suggestion by checking the edits map to reverse-map the name.

The hook does NOT use React Query for these fetches -- it's a custom lifecycle that doesn't fit the standard query pattern. It uses plain fetch + useEffect for polling, matching the pattern shown in the research document.

Import `useQueryClient` from `@tanstack/react-query` but do NOT call `invalidateModelQueries` in the hook -- leave that to the dialog component after `apply()` resolves (the dialog knows the entity type and can invalidate the right model + Tags).
