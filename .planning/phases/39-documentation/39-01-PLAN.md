### Section 1: "## Per-Feature LLM Overrides"

Cover:

1. **What it does** — Project admins can override which LLM integration is used for specific AI features (test case generation, markdown parsing, smart test case selection, editor writing assistant, connection test, QuickScript, and auto-tagging). This is configured per project on the AI Models settings page.

2. **How to configure** — Step-by-step:
   - Navigate to **Project Settings** > **AI Models**
   - The "Per-Feature LLM Overrides" card shows a table with columns: Feature, Override, Effective LLM, Source
   - For any feature row, select an LLM integration from the Override dropdown to assign a specific integration for that feature in this project
   - Click the X button next to the dropdown to clear the override and fall back to the next level in the resolution chain
   - The "Effective LLM" column shows which integration will actually be used
   - The "Source" column shows a color-coded badge indicating where the effective integration comes from:
     - **Project Override** (blue) — a per-feature override is set on this project
     - **Prompt Config** (gray) — the prompt configuration has a per-prompt LLM assignment
     - **Project Default** (outline) — the project's default integration is being used
     - **No LLM configured** (red) — no integration is available at any level

3. **Use cases:**
   - Use a fast, cheap model (e.g., GPT-3.5) for connection tests and a powerful model (e.g., GPT-4) for test case generation
   - Override a single feature to use a different provider without changing other features
   - Project teams can customize their LLM usage independently of the system-wide prompt configuration

### Section 2: "## LLM Resolution Chain" {#llm-resolution-chain}

Document the three-level resolution chain that determines which LLM integration is used when an AI feature is invoked:

1. **Project Feature Override** (highest priority) — If the project has a per-feature LLM override set via LlmFeatureConfig for this specific feature, that integration is used.
2. **Prompt Configuration Assignment** — If the resolved prompt (from the project's or system default prompt config) has a per-prompt LLM integration assigned, that integration and optional model override are used.
3. **Project Default Integration** (lowest priority) — The project's default LLM integration is used as the final fallback.

Include a simple text diagram or table:

```
Prompt has per-prompt LLM assigned? --> YES --> Use prompt's integration (+ model override if set)
  |
  NO
  v
Use project default integration
```

Add a note: "This resolution chain applies independently per AI feature. Different features in the same project can resolve to different LLM integrations depending on what is configured at each level."
