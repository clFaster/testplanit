```typescript
export type EntityType = "repositoryCase" | "testRun" | "session";

// Represents one entity's content extracted for LLM consumption
export interface EntityContent {
  id: number;
  entityType: EntityType;
  name: string;
  textContent: string;       // All content flattened to plain text
  existingTagNames: string[]; // Tags already on this entity (to avoid re-suggesting)
  estimatedTokens: number;   // chars / 4 estimate
}

// A single tag suggestion from the AI
export interface TagSuggestion {
  entityId: number;
  entityType: EntityType;
  tagName: string;           // Normalized tag name
  isExisting: boolean;       // true = matches an existing project tag, false = new tag
  matchedExistingTag?: string; // If fuzzy-matched, the exact existing tag name
  confidence?: number;       // Optional confidence from AI (if provided)
}

// Result for one batch of entities
export interface BatchAnalysisResult {
  suggestions: TagSuggestion[];
  tokensUsed: number;
}

// Full result across all batches
export interface TagAnalysisResult {
  suggestions: TagSuggestion[];
  totalTokensUsed: number;
  batchCount: number;
  entityCount: number;
}

// Configuration for batching
export interface BatchConfig {
  maxTokensPerRequest: number; // From LlmProviderConfig
  contentBudgetRatio: number;  // 0.6-0.7 — portion of token budget for entity content
  systemPromptTokens: number;  // Estimated tokens for system prompt + tag list
}

// Raw AI response shape (what we expect back from the LLM)
export interface AutoTagAIResponse {
  suggestions: Array<{
    entityId: number;
    tags: string[];
  }>;
}
```

2. Create `content-extractor.ts` with these exported functions:

**`extractTiptapText(json: unknown): string`** — Recursively walks Tiptap JSON structure (nodes with `type`, `content` array, and leaf `text` nodes). Returns concatenated plain text with spaces between blocks. Handle null/undefined input gracefully (return ""). Handle string input (return as-is — some fields may already be plain text).

**`extractEntityContent(entity, entityType, folderPath?): EntityContent`** — Takes a raw entity object (from Prisma query with includes) and extracts all relevant content into `EntityContent`:

For `repositoryCase`:
- `name` from entity.name
- `textContent` = join of: folder path (if provided), name, each step's `extractTiptapText(step.step)` + `extractTiptapText(step.expectedResult)`, each caseFieldValue's `field.name: extractFieldValue(cfv)` (skip empty values)
- `existingTagNames` from entity.tags.map(t => t.name)

For `testRun`:
- `textContent` = join of: name, `extractTiptapText(note)`, `extractTiptapText(docs)`
- `existingTagNames` from entity.tags

For `session`:
- `textContent` = join of: name, `extractTiptapText(note)`, `extractTiptapText(mission)`, each sessionFieldValue's `field.name: extractFieldValue(sfv)`
- `existingTagNames` from entity.tags

**`extractFieldValue(fieldValue): string`** — Helper to extract display value from CaseFieldValues/SessionFieldValues. The `value` is Json — could be string, number, boolean, array of strings (multiselect), or object. For arrays, join with ", ". For objects, JSON.stringify. For null/undefined, return "".

Set `estimatedTokens = Math.ceil(textContent.length / 4)` per existing pattern.
