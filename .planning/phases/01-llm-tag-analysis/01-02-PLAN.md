**`matchTagSuggestions(aiSuggestedTags: string[], existingProjectTags: string[], entityExistingTagNames: string[]): Array<{ tagName: string; isExisting: boolean; matchedExistingTag?: string }>`**

Algorithm:
1. Build a normalized map of existing project tags: `Map<normalizedName, originalName>`
2. For each AI-suggested tag:
   a. Normalize it
   b. Skip if it matches (normalized) any tag in `entityExistingTagNames` (entity already has it)
   c. Check exact match against existing project tags map -> isExisting=true, matchedExistingTag=original
   d. If no exact match, check fuzzy: iterate existing tags, compare using simple Levenshtein distance. Match if distance <= 2 for tags <= 10 chars, or one normalized name is a substring of another (handles "auth" matching "authentication"). Take the closest match.
   e. If still no match -> isExisting=false, tagName=AI's suggestion (trimmed)

**`async analyzeTags(params: { entityIds: number[], entityType: EntityType, projectId: number, userId: string }): Promise<TagAnalysisResult>`**

Orchestration flow:
1. Fetch LLM integration ID via `LlmManager.getDefaultIntegration()`. If none, throw descriptive error.
2. Fetch `LlmProviderConfig` for the integration to get `maxTokensPerRequest`.
3. Fetch all existing project tags: `prisma.tags.findMany({ where: { isDeleted: false } })` — note Tags model has no projectId, tags are global. This is correct per schema.
4. Resolve prompt via `PromptResolver.resolve(LLM_FEATURES.AUTO_TAG, projectId)`.
5. Fetch entities based on `entityType`:
   - `repositoryCase`: `prisma.repositoryCases.findMany({ where: { id: { in: entityIds }, isDeleted: false }, include: { steps: { where: { isDeleted: false }, orderBy: { order: "asc" } }, caseFieldValues: { include: { field: true } }, tags: true, folder: true } })`. Then build folder path by querying `RepositoryFolders` ancestors (walk parentId up to root). Be mindful of ZenStack alias length — keep includes shallow (max 2 levels).
   - `testRun`: `prisma.testRuns.findMany({ where: { id: { in: entityIds }, isDeleted: false }, include: { tags: true } })`
   - `session`: `prisma.sessions.findMany({ where: { id: { in: entityIds }, isDeleted: false }, include: { sessionFieldValues: { include: { field: true } }, tags: true } })`
6. Convert each entity to `EntityContent` via `extractEntityContent()`.
7. Build batches using `createBatches(entities, batchConfig)`.
8. Process batches sequentially: for each batch, build user prompt with entity data, call `LlmManager.chat()`, parse JSON response, apply `matchTagSuggestions()`.
9. Aggregate results into `TagAnalysisResult`.

**`createBatches(entities: EntityContent[], config: BatchConfig): EntityContent[][]`** (exported for testing)

Algorithm:
- Calculate content budget: `config.maxTokensPerRequest * config.contentBudgetRatio - config.systemPromptTokens`
- For each entity:
  - If entity.estimatedTokens > contentBudget: truncate textContent to fit (slice string to `contentBudget * 4` chars), recalculate estimatedTokens, put in own batch
  - Otherwise: add to current batch if sum of tokens fits, else start new batch
- Default `contentBudgetRatio`: 0.65
- Estimate `systemPromptTokens`: `Math.ceil(resolvedPrompt.systemPrompt.length / 4) + Math.ceil(existingTagsString.length / 4)` where existingTagsString is the comma-joined tag list

**`buildUserPrompt(entities: EntityContent[], existingTagNames: string[]): string`** (private)

Format:
```
EXISTING PROJECT TAGS:
tag1, tag2, tag3, ...

ENTITIES TO ANALYZE:

--- Entity 1 (ID: 123, Type: repositoryCase) ---
Name: Login test
Already tagged: [smoke, regression]
Content:
{textContent}

--- Entity 2 (ID: 456, Type: repositoryCase) ---
...
```

**Error handling:** Wrap `JSON.parse` of LLM response in try/catch. If parsing fails, log warning and return empty suggestions for that batch. If LLM call fails, log error and skip batch (don't fail entire operation).
