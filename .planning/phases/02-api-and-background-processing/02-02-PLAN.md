Create `testplanit/app/api/auto-tag/apply/route.ts`:

- Export async POST handler.
- Auth: `getServerSession(authOptions)`. Return 401 if no session.
- Validate body with zod:
  ```typescript
  const applySchema = z.object({
    suggestions: z.array(z.object({
      entityId: z.number(),
      entityType: z.enum(["repositoryCase", "testRun", "session"]),
      tagName: z.string().min(1).max(255),
    })).min(1),
  });
  ```
  Per user decision: frontend sends only accepted suggestions (subset), not full set with accept/reject flags.

- Import prisma from `@/lib/prisma` (the default client, not enhanced -- this is a direct DB operation).

- Execute within `prisma.$transaction()` for all-or-nothing behavior per user decision:

  a. **Deduplicate tag names**: Extract unique tag names from suggestions array.

  b. **Upsert all tags upfront** (avoids race condition per Pitfall 3 in research):
     ```typescript
     const tagMap = new Map<string, number>();
     for (const name of uniqueTagNames) {
       const tag = await tx.tags.upsert({
         where: { name },
         create: { name },
         update: {},  // no-op if exists
       });
       tagMap.set(name, tag.id);
     }
     ```

  c. **Connect tags to entities**: For each suggestion, use the appropriate model based on entityType:
     - `"repositoryCase"` -> `tx.repositoryCases.update({ where: { id }, data: { tags: { connect: { id: tagId } } } })`
     - `"testRun"` -> `tx.testRuns.update({ where: { id }, data: { tags: { connect: { id: tagId } } } })`
     - `"session"` -> `tx.sessions.update({ where: { id }, data: { tags: { connect: { id: tagId } } } })`

  d. Handle the case where a tag is already connected to an entity (connecting an already-connected tag should be a no-op -- Prisma/ZenStack handles this gracefully for many-to-many `connect`).

- Return response:
  ```typescript
  {
    applied: suggestions.length,
    tagsCreated: newTagCount,  // count of tags that were created (not pre-existing)
    tagsReused: reusedCount,   // count of tags that already existed
  }
  ```
  To compute these: before the transaction, query existing tag names. After upsert loop, compare with the set.

- Error handling:
  - Catch transaction errors. If ZenStack/Prisma error, return 500 with `{ error: "Failed to apply tags" }`.
  - Use `isUniqueConstraintError` from `~/lib/utils/errors` as a safety net if upsert somehow still races (unlikely but defensive).
  - If any entity ID is not found (update fails), the transaction rolls back automatically. Return 400 with `{ error: "One or more entities not found" }`.

- Wrap the whole handler in try/catch for unexpected errors. Return 500.
